# OfferScope / 录取雷达

AI 留学申请诊断网站 — **找中介之前，先看清你的真实申请位置。**

一个面向中国学生的去中介化 AI 留学申请诊断工具。用户选择本科或硕士、目标地区/学校/专业，填写背景，回答 2–3 个 AI 关键追问，即可获得免费初测；付费后解锁完整诊断报告并可打印/下载 PDF。

## 支付（ZPay 微信/支付宝，易支付标准）

真实收款已接入 ZPay。把商户信息填入 `.env`：

```
ZPAY_PID=你的商户ID
ZPAY_KEY=你的商户密钥
ZPAY_HOST=https://z-pay.cn
SITE_URL=https://offerscope.org   # 部署后填正式域名；本地用 http://localhost:3000
```

流程：结算页选微信/支付宝 → 后端 `/api/pay/create` 下单并跳转 ZPay 收银台扫码 → ZPay 异步回调 `/api/pay/notify`（MD5 验签 + 金额校验，返回 `success`）→ 前端轮询 `/api/pay/status` 检测到已付 → 自动解锁报告。订单存于 `orders.json`（已 gitignore）。

⚠️ ZPay 异步回调需要**公网可访问**，因此真实到账要在部署到 `offerscope.org` 之后才完整生效。本地未配置/无法收到回调时，结算页会显示「模拟支付成功（测试）」按钮，用于本地把流程点通。

## 当前实现（可直接运行的原型）

这是一个**单文件、零依赖**的可点击原型（`index.html`），实现了 MVP 文档中的完整闭环，开箱即用、无需 API Key、无需后端：

1. 首页（Hero / 痛点 / 流程 / 报告预览 / 价格 / 免责声明）
2. 选择本科 / 硕士（动态表单、追问、评分、报告随之变化）
3. 目标地区 / 学校 / 专业（仅支持美国、英国、加拿大、澳洲、新加坡、中国香港）
4. 动态背景表单（本科与硕士字段完全不同）
5. AI 关键追问（按申请阶段与专业动态生成，每题附"为什么问"）
6. 免费初测结果（总体等级、风险等级、学校初步判断、主要风险、付费引导）
7. Mock 支付（¥19 标准版 / ¥49 专业版，含成功/失败页）
8. 完整诊断报告（五维雷达图、评分拆解、逐校分析、冲刺/匹配/保底分层、优势短板、风险、30/60/90 天路线、文书策略、家长可读总结、行动清单、中介名单复核、专业版扩展）
9. 浏览器打印 / 下载 PDF（Print CSS）
10. 我的报告（localStorage 历史记录）

### 评分引擎

基于 MVP 文档的评分权重，规则化（确定性）生成结果，无需调用外部 AI：

- 本科：学术成绩与课程难度 30 / 语言与标化 20 / 活动与奖项 20 / 个人主线与专业匹配 15 / 申请策略与文书空间 15
- 硕士：学术背景 30 / 语言与标化 15 / 专业匹配度 20 / 实习科研项目 20 / 申请材料与策略 15

学校判断标签严格使用：高风险冲刺 / 冲刺 / 匹配偏冲刺 / 匹配 / 匹配偏稳 / 保底 / 不建议申请。
全站不出现"保证录取/必录/稳录/100%/录取概率百分比"等表述。

## 如何运行

### 方式一：纯前端（最简单，模拟 AI）

直接用浏览器打开 `index.html` 即可，无需安装任何依赖。此时 AI 由内置规则引擎生成（确定性、秒出、免费）。

```bash
open index.html                 # macOS
```

### 方式二：接入真实 DeepSeek（推荐，需要 Key）

加了一个**零依赖的本地服务器**（`server.js`），把 DeepSeek 安全地接在服务器端——你的 API Key 只存在服务器的 `.env` 里，**绝不会暴露给浏览器**。

```bash
# 1. 复制环境变量模板并填入你的 key
cp .env.example .env
#    然后用编辑器打开 .env，把 DEEPSEEK_API_KEY 改成你的真实 key

# 2. 启动（需要 Node 18+）
node --env-file=.env server.js
#    或： npm start

# 3. 浏览器访问
#    http://localhost:3000
```

启动后终端会显示 `DeepSeek key：已配置 ✓`。如果 DeepSeek 调用失败（key 错、网络不通、返回格式异常），**网站会自动回退到规则引擎**，用户永远拿得到结果，不会白屏。

**AI 的分工（混合架构）**：分数、学校冲刺/匹配/保底分层、报告结构由规则引擎确定（稳定、合规、不乱编分数）；DeepSeek 负责它最擅长的**中文叙述**——追问问题、初测文案、执行摘要、优势短板、文书策略、家长可读总结等。这样既发挥 DeepSeek 中文优势，又保证报告不会出现"录取概率"之类的违规表述。日后可逐步把更多部分交给模型。

`.env` 里还可以改模型名和接口地址（`DEEPSEEK_MODEL_FAST` / `DEEPSEEK_MODEL_PRO` / `DEEPSEEK_BASE_URL`），DeepSeek 若调整命名，改这里即可，无需动代码。
